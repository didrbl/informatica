План тестирования:

1. Надо протестировать корректность работы методов класса Queue, в т.ч.:
    * метод enqueue лобавляет элемент в левый конец списка
    * метод deque убирает элемент из правого конца списка, если список не пустой
    * методы front и back показывают данные элементов в правом и левом конце списка соответственно, если в них есть
    элементы, иначе - None
    * метод clear убирает все элементы из списка
    * метод size показывает число элементов в списке

2. Будем тестировать то, что надо протестировать

3. Будем тестировать с помощью пакета unittest, написав класс для тестирования, наследующий от unittest.TestCase и
имеющий методы для тестирования одного или нескольких методов Queue. Некоторые методы будем тестировать совместно,
так как для этого используется один и тот же код^
    * для тестирования enqueue,front, back, size вызовем enqueue 2 раза и проверим, что порядок елементов в объекте
    совпадает с ожидаемым, front и back показывают правый и левый конец списка элементов, size - число элементов
    * для тестирования dequeue добавим в экземпляр Queue 4 элемента, а потом удалим два, после чего проверим корректность
    списка элементов
    * для тестирования clear добавим 4 элемента в экземпляр Queue, а затем вызовем метод clear и проверим, что список не
    содержит элементов
    * для тестирования корректности поведения Queue в случае 1 элемента элементов добавим в его экземпляр
    1 элемент и проверим, что результаты вызовов front и back совпадают.
    * для тестирования корректности поведения Queue в случае отсутствия элементов вызовем метод dequeue по отношению к
    пустому экземпляру Queue.
